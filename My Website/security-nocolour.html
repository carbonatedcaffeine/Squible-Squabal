Fuchsia security

The kernel isolates processes by default, and grants permissions and hardware resources explicitly. The kernel also passes permissions and resources by handles rather than name, which “leads to a system that only grants software access to what it needs”.

Nearly all software running in the userspace on Fuchsia is a component. These are from system services, to end user software. “The component framework encourages the composition of loosely coupled software. The capabilities/permissions used must be “explicitly” declared.

Packages come with all the files needed to run them, “packages are a collection of components, files and metadata” and are isolated from each other and only “has visibility to it’s own package”.

"Fuchsia aims to have no ambient authority, which means every operation is scoped to an object capability. Similarly, Fuchsia has no global file system. Instead, each program is given its own local namespace in which to operate."
